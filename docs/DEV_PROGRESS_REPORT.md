# 개발 완성도 리포트 (주기 실행)

## 의미

- **백엔드**: 공개·관리자 API, 인증, 결제 웹훅 골격, 스모크, 오류 형식 등 **서버가 동작 가능한 수준**.
- **DB·영속성**: Prisma 스키마, 파일 폴백, 감사·CMS·관심종목 등 **데이터 저장 구조**.
- **프론트엔드**: 페이지·`oneaiFetch` 연동 비율, 마이페이지·관리자 등 **실사용 UI 진행도**.

**전체 %**는 위 세 축을 **34% / 33% / 33%** 가중 평균한 값입니다.  
“진짜 서비스 오픈”에는 보안 감사, 실데이터·실결제 검증, 운영 모니터링 등이 추가로 필요합니다.

## 실행

프로젝트 루트에서:

```bash
npm run progress
```

한 줄 로그를 파일로 누적:

```bash
npm run progress:log
```

## 1시간마다 자동 실행

에이전트는 상시 실행되지 않으므로, **로컬 또는 CI 작업 스케줄러**로 위 명령을 걸어 두세요.

### Windows (작업 스케줄러)

1. 작업 스케줄러에서 **기본 작업 만들기**
2. 트리거: **매일** 또는 **한 시간마다** (고급 설정에서 반복 간격 1시간)
3. 동작: 프로그램 시작  
   - 프로그램: `powershell`  
   - 인수: `-NoProfile -Command "cd 'C:\Users\USER\stock'; npm run progress:log"`

경로는 본인 PC 기준으로 수정합니다.

### GitHub Actions (선택)

저장소에 워크플로를 두고 `cron: '0 * * * *'`(매시 정각)으로 `npm run progress`만 실행·아티팩트 저장할 수 있습니다. (레포가 비공개여도 무료 한도 내에서 가능.)

## 수치 조정

`scripts/dev-progress-report.js` 상단의 **`BASELINE`**에서 수동 기준을 바꿉니다.  
자동 스캔 비율은 **`autoWeight`**로 조절합니다 (높을수록 파일 존재·간단 휴리스틱 비중 증가).

## 이력 파일

`npm run progress:log` 사용 시 `docs/progress-history.log`에 타임스탬프별 리포트가 append 됩니다. Git에 올리지 않으려면 `.gitignore`에 추가하세요.
